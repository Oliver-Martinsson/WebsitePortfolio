Overview

A full-stack web application for publishing and managing blog posts. Built with ASP.NET Core Razor Pages, it delivers a complete content management experience: a public-facing reader view and a password-protected admin panel for authors.

User accounts are handled by ASP.NET Identity, which provides secure password hashing, login/logout flows, and role management out of the box.

Key Features

  • Public blog listing with pagination and tag-based filtering
  • Markdown post editor with live preview in the admin panel
  • User registration and login via ASP.NET Identity
  • Role-based access: only authenticated admins can create or delete posts
  • Tag management system — add, edit, and browse posts by tags
  • EF Core Code First with SQL Server for data persistence

Architecture

Razor Pages naturally organizes each page into a paired .cshtml view and a PageModel code-behind, keeping the application tidy. Business logic is extracted into service classes, and EF Core handles all database interactions through a repository layer.

What I Learned

This project gave me strong familiarity with ASP.NET Identity, Razor Pages conventions, and tag-based content organization. Implementing a markdown renderer and managing image uploads also introduced me to file handling in ASP.NET Core.
