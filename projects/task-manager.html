Overview

A Windows desktop application for managing personal tasks and to-dos. Built using WPF with the MVVM (Model-View-ViewModel) pattern to keep the UI logic cleanly separated from business logic and data.

All data is persisted locally using SQLite via Entity Framework Core, so the app works fully offline without any server dependency.

Key Features

  • Create, edit, and delete tasks with titles, descriptions, and due dates
  • Priority levels (Low, Medium, High) with color-coded indicators
  • Category grouping and filtering for organized task views
  • Mark tasks as complete with visual strike-through and progress tracking
  • SQLite local storage — data persists between sessions
  • Responsive layout adapting to different window sizes

Architecture

The MVVM pattern was strictly followed: Views are pure XAML with no code-behind logic, ViewModels expose observable properties and commands, and Models represent the data layer. Data binding and INotifyPropertyChanged keep the UI in sync automatically.

What I Learned

This project gave me solid hands-on experience with WPF data binding, MVVM, and the ICommand pattern. Managing real-time UI updates with ObservableCollection and understanding the WPF dispatcher were the most valuable takeaways.
