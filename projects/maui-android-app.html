TrackIt — .NET MAUI Android App

A cross-platform mobile application built with .NET MAUI, targeting Android. Consumes a public REST API to display real-time data, with a focus on Android API compatibility, offline caching, and a clean MVVM architecture.

View on GitHub

Overview

TrackIt is a .NET MAUI application targeting Android (API 24+) that lets users browse and track items fetched from a REST API. The app was built with Android compatibility as the primary concern — handling permission requests, back-navigation gestures, and hardware acceleration differences between API levels 24 through 34.

Data is fetched asynchronously using HttpClient with a typed client pattern, deserialized with System.Text.Json, and cached locally with Preferences and SQLite-net-pcl so the app remains usable when the device is offline.

Key Features

  • Live REST API integration with async/await and typed HttpClient
  • Offline-first caching using SQLite-net-pcl and MAUI Preferences
  • Android back-navigation support via Shell and hardware back button handling
  • Runtime permission requests (network state, storage) for Android API 23+
  • Pull-to-refresh and infinite scroll for large datasets
  • Light/dark theme following the Android system setting
  • Tested on Android API 24 (Nougat) up to API 34 (Android 14)

Android Compatibility

One of the core challenges was ensuring consistent behaviour across Android versions. Key areas addressed:

  • AndroidManifest.xml — declared INTERNET and ACCESS_NETWORK_STATE permissions and set android:minSdkVersion="24" with targetSdkVersion="34".
  • Network security config — applied a custom network_security_config.xml to enforce HTTPS and handle cleartext traffic rules for older API levels.
  • Scoped storage — used MediaStore APIs for file writes on Android 10+ (API 29) while falling back to legacy external storage for API 24–28.
  • WindowInsets / edge-to-edge — handled system bar insets to avoid content being clipped behind the navigation bar on API 30+.

Architecture

The project follows MVVM using the CommunityToolkit.Mvvm source generators ([ObservableProperty], [RelayCommand]) to eliminate boilerplate. ViewModels are registered with the MAUI DI container via MauiProgram.cs, keeping every layer independently testable.

What I Learned

Building for Android specifically taught me how platform-specific behaviour in MAUI differs from plain Xamarin or Windows targets. Navigating the Android lifecycle, handling back-stack correctly inside MAUI Shell, and writing API-level conditional code using OperatingSystem.IsAndroidVersionAtLeast() were the most valuable takeaways from this project.
