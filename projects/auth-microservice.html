Overview

A self-contained authentication microservice designed to be plugged into any larger system. It handles user registration, login, token issuance, and token refresh — all as a standalone service that can be deployed independently via Docker.

The service implements an OAuth2-inspired flow using short-lived access tokens (JWT) and long-lived refresh tokens stored server-side, enabling seamless session renewal without re-authentication.

Key Features

  • JWT access tokens with configurable expiry and signing keys
  • Refresh token rotation — new token issued on every refresh
  • Per-IP rate limiting on login and registration endpoints
  • Structured logging with Serilog (console + file sinks)
  • Fully containerized with Docker and docker-compose
  • Health check endpoint for container orchestration compatibility

Architecture

The service is built as a minimal ASP.NET Core API with a single responsibility: authentication. It exposes only a handful of endpoints and has zero UI. The refresh token store uses EF Core with SQL Server, and all secrets are injected via environment variables — never hard-coded.

What I Learned

This project deepened my understanding of token security, refresh token rotation strategy, and why short-lived access tokens matter. Containerizing with Docker and writing a proper docker-compose setup for the service and its database was also a major learning milestone.
